# Plan: Package ai-context as a Distributable npm CLI (v1.0.0)

## Problem / Goal
Users must clone the repo or run `scripts/ai-context.sh` manually to install AI Context. This requires knowing the repo URL, having bash, and running a script. There is no discoverable, zero-install way to get started. Packaging as an npm CLI enables `npx ai-context init` with no prerequisites, cross-platform support, and a better interactive experience.

## Approach
Build a TypeScript CLI under `packages/cli/` that bundles all template files and replicates all bash installer behavior. Bash scripts are frozen at v0.5.0 as a legacy option — all new development lives in TypeScript. The CLI is published to npm as `ai-context` (verify name availability; fallback: `ai-context-cli`).

Key UX additions over the bash scripts:
- Interactive agent adapter selection at `init` time (choose which adapter files to install: Claude, Cursor, Codex, Copilot, Antigravity)
- Interactive CLI agent selection for project configuration (shared by `init` and `setup`)
- CLI health check: two-step verification (binary exists + authenticated/responsive) with spinner
- Real-time streaming output from CLI agents during setup (stream-json for claude, generic for others)
- Config-driven CLI registry in `agentCLI.ts` — adding a new CLI agent is one config entry
- `ai-context setup` command that runs the same interactive setup flow as `init`
- `ai-context status` for diagnostics (installed version, schema, apply mode, agent adapters)
- `--gitignore` flag to add sessions/ and backups/ to `.gitignore`
- Eliminates the `.ai-context-setup/` folder from installed targets

## Key Decisions
- **Language**: TypeScript (Node ≥18) — better JSON handling, cross-platform, npm-native
- **No parallel bash maintenance**: all core logic moves to TypeScript; bash scripts frozen at v0.5.0
- **Windows support**: macOS + Linux + Windows. `fs/promises` + `path.join()` throughout. `agentCLI.ts` uses `where` on Windows, `which` on Unix. Claude Code `.sh` hooks are a pre-existing bash limitation on Windows — out of scope.
- **Template bundling**: static copy in `packages/cli/src/templates/` (bundled in npm package); `scripts/sync-templates.sh` syncs from repo root before publish
- **Schema bump**: manifest schema v4 → v5 (new field: `agents_installed: string[]`)
- **managed_by**: npm CLI writes `"npm:ai-context@1.0.0"`; bash writes `"scripts/ai-context.sh"` — both accepted
- **`.ai-context-setup/` eliminated**: prompt templates live in `src/setup-prompts/` inside the npm package, not copied to target projects
- **Plans directory**: this plan is the first entry in `.ai-context/plans/`

## Implementation Notes

### Package structure
```
packages/cli/
├── package.json              # name: "ai-context", version: "1.0.0"
├── src/
│   ├── index.ts              # commander entrypoint
│   ├── commands/             # init, apply, setup, uninstall, status, version
│   ├── core/                 # manifest, applyMode, backup, restore, copyTemplates,
│   │                         # claudeHooks, ownership, legacyStandards, gitignore, agentCLI
│   ├── prompts/              # agentSelector, targetDir
│   ├── templates/            # bundled template files
│   ├── setup-prompts/        # fresh-install.md, upgrade.md (not copied to target)
│   └── ui/                   # logger (picocolors), spinner
```

### Dependencies
- `commander` — CLI framework (zero deps)
- `@inquirer/prompts` — ESM-native interactive prompts
- `picocolors` — colorized output (auto-disables in CI)
- No `fs-extra`, no `glob` — `fs/promises` is sufficient

### Core module order (implement in this sequence)
`ownership` → `manifest` → `applyMode` → `backup` → `legacyStandards` → `restore` → `claudeHooks` → `gitignore` → `copyTemplates` → `agentCLI` → `setupFlow`

### Commands
- `init [path]` — interactive; adapter selection (multi-select: which adapter files to install); auto-runs interactive setup after install (single-select: which CLI agent to configure project)
- `apply <path>` — non-interactive bash parity; installs all agents
- `setup [path]` — interactive CLI agent selection → health check → run setup prompt. Same flow as init's post-install step. `--cli <name>` to skip picker, `--print` to just print prompt
- `uninstall <path>` — removes managed files; properly cleans hooks from `settings.json`
- `status [path]` — shows installed version, schema version, apply mode, agent adapters present
- `version` — prints CLI version and schema version

### Architecture: two concerns, cleanly separated
1. **Adapter installation** (`init`) — which agent adapter files to copy (Claude, Cursor, Codex, Copilot, Gemini). Multi-select checkbox.
2. **Project configuration** (`interactiveSetup` in `setupFlow.ts`) — which CLI agent runs the setup prompt. Single-select from `CLI_REGISTRY`. Shared by both `init` (after file copy) and `setup` (standalone).

### CLI Registry (`agentCLI.ts`)
Config-driven — each CLI agent is a `CLIConfig` entry with: `bin`, `pingArgs`, `promptArgs`, and optional `streaming` config. Adding a new CLI (e.g. gemini) is one registry entry. All agents get the same health check, spinner, streaming, and fallback behavior.

### Critical source files to port from
- `scripts/ai-context.sh` — complete behavioral spec
- `scripts/uninstall-ai-context.sh` — uninstall behavioral spec
- `tests/test-ai-context-installer.sh` — 8 integration scenarios to replicate in TypeScript
- `.ai-context-setup/SETUP-PROMPTS.md` — content to split into `src/setup-prompts/*.md`
